Labels & Tags

Two families: Status tags for system states and feature badges (always uppercase), and Content tags for categories and descriptive labels (always sentence case).

Status & Badge Tags — always UPPERCASE

Use for: system states (Active, Beta, Deprecated), plan levels (Free, Pro), feature flags (New, Coming soon). Short 1–3 word labels only.

Content & Category Tags — always Sentence case

Use for: blog categories, article types, content classification, resource labels. Descriptive, human-readable. Can be multi-word.

Product Case study Guide Automation Customer story Free · Open source
Section Overline variant — content tag, sized for landing pages

Use for: section headings on landing pages where the tag sits above a large H2 and needs more visual weight. Keeps sentence case and soft colours — distinguished from Status tags by not being uppercase/bold. Add .c-blog-tag-section alongside the base class.
