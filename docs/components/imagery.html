Imagery Style Guide

Without a dedicated illustrator, Landbot uses a 3-tier system that produces a consistent, high-quality visual language entirely in-house — no custom illustration required.

The 3-Tier System

Tier 1 · Primary
Product Screenshots
Your most credible asset. Show the actual Landbot builder in browser or phone device frames. Works for hero, features, and case studies.
😊
🙂
Tier 2 · Contextual
People Photography
Real humans in modern work contexts — for testimonials, team pages, and case studies. Carefully curated from free stock sources.
Tier 3 · Decorative
Abstract Brand Visuals
Gradient blobs, glow effects, and geometric shapes built in Figma using brand tokens. Used as section backgrounds — never as storytelling.
When to use which tier
Tier 1 · Product
✦ Hero sections ✦ Feature breakdowns ✦ How it works ✦ Case study visuals ✦ Integration pages
Tier 2 · People
✦ Testimonial cards ✦ Customer logos section ✦ About / Team page ✦ Blog author avatars ✦ Case study headers
Tier 3 · Abstract
✦ Section backgrounds ✦ Empty states ✦ 404 / error pages ✦ Dark section accents ✦ Social media cards

Free sources (curated, brand-safe):
• Unsplash (unsplash.com) — best overall quality, large library • Pexels (pexels.com) — good diversity, strong workspace category • Moose by Icons8 (icons8.com/photos) — consistent style, easy to match shots
⚠ Before publishing: Check the licence for every photo. Unsplash and Pexels are free for commercial use but require no model releases — avoid using identifiable people in contexts that imply endorsement.

Product Screenshots

Show the product inside a device frame — this gives visual context and elevates perception. Never show a raw screenshot floating on a background.

Rules • Always use the same screenshot scale within a section — mixing zoom levels looks inconsistent. • Remove any personal data, customer names, or internal content before screenshotting. • Use the actual Landbot UI — never fake or redesign the interface for marketing. • For feature rows, show the most visually interesting part of the UI cropped tightly.

Illustrations (When Needed)

Recommendation: Landbot's strong brand color system means you rarely need custom illustrations. Product shots + abstract visuals carry most pages. Use illustrations only for empty states, onboarding tooltips, and 404/error pages — where there is no product UI to show.

When you do need illustrations, use these free libraries — all support custom brand color input:

🎨
unDraw
Free · Open source
SVG illustrations with a single brand-color input. Clean, flat style. 1000+ scenes. Best for empty states and feature explainers.
undraw.co
✏️
Storyset
Free · Animated
Customisable colour + animated SVGs. Modern, SaaS-appropriate style. Has a Figma plugin. Great for hero explainer sections.
storyset.com
🧩
Blush
Freemium
Character-based illustrations with a Figma plugin. High diversity. Colour-customisable. Best for testimonial and human-context sections.
blush.design
Illustration usage rules
RuleReason
Always set the illustration's primary color to #5A60F6Ties it to the brand, avoids generic stock look
Use one library per project — never mix stylesInconsistent styles break visual cohesion
Max 1 illustration per page sectionMore than one competes for attention
Never use illustrations as hero imagesProduct screenshots are more credible for B2B SaaS
Export as SVG — never PNG for web illustrationsSVG scales without blur at all screen sizes
Match illustration complexity to text densitySimple section = simple illustration; detailed section = no illustration

Do / Don't

✓ Do
✓Show the real Landbot UI inside a clean device frame
✓Use people photos that feel natural and unposed
✓Apply brand gradients as section backgrounds and accents
✓Use one illustration library consistently across the whole site
✓Crop and frame images intentionally — leave room for text overlay
✓Set illustration primary color to #5A60F6 always
✓Pair dark sections with glow effects for visual depth
✗ Don't
✗Use floating screenshots with no device frame or background context
✗Mix illustration styles from different libraries on the same page
✗Use corporate stock photos (suits, handshakes, pointing at whiteboards)
✗Use illustrations as the primary hero visual — use product instead
✗Apply heavy filters or black & white to photography
✗Show personal data or customer info in product screenshots
✗Use stock art of robots or chat bubbles — it's generic for a chatbot brand
