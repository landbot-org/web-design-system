Usage Rules
✓ Do
  • Use brand gradient on CTA buttons and 1–3 bold (700) accent words in a headline
  • Use dark gradient for full-bleed dark sections — white text is always safe on it
  • Layer both glow gradients over the dark gradient for a rich hero
  • Use purple soft as a section background tint — standard text colors apply
  • Use gradient borders (2px gradient outline) to highlight a card
✗ Don't
  • Don't use brand gradient as a solid section background — contrast at the pink end is only 3.4:1 with white text
  • Don't use gradient text at regular weight (400) — thin strokes kill the effect; bold (700) only
  • Don't apply gradient text below 32px — too small to read the color shift
  • Don't use more than two gradient-heavy sections back to back
  • Don't invent new gradient combinations — use only the five tokens above

Rules
✓ Do
  • Animate only transform and opacity — never width, height, margin, or padding
  • Use ease-out for almost everything — it feels instant and natural
  • Stagger sequential elements 60–80ms each (3 cards: 0ms, 60ms, 120ms)
  • Respect prefers-reduced-motion — skip all scroll animations if set
✗ Don't
  • Don't use linear easing — it looks mechanical
  • Don't exceed 600ms for any UI interaction — it feels broken
  • Don't animate error messages, focus rings, or critical state changes
  • Don't animate more than 3–4 elements simultaneously in the same area

Buttons

Five button variants forming a clear visual hierarchy. Always use the highest-priority variant that fits the context — never two Primary buttons on the same view.

Variants & Hierarchy
1 Primary
.cta-primary
The hero conversion action. Pink→purple gradient, always white text. Use once per page for the single most important CTA — Start free trial, Get started, Request demo.
2 Brand
.cta-purple
Solid brand purple for important secondary actions. Use when a gradient button would be too loud, or as the primary CTA in smaller sections (pricing cards, feature callouts).
3 Outlined
.cta-outline
Purple border, transparent background. Used alongside a Primary or Brand button as a lower-emphasis alternative — "Sign in" next to "Get started free", secondary actions in hero sections.
4 Subtle
.cta-alt-purple
Light purple tint with dark purple text. For low-emphasis actions in dense layouts — feature card links, inline CTAs inside content blocks, "View example" type actions.
5 On Dark
.cta-white
White background with navy text. Used exclusively on dark/navy section backgrounds as a secondary CTA. Always paired with a Primary (gradient) button as the lead action.
Text Links
.cta-textlink
Pink accent — "Learn more" links, editorial inline links. No underline at rest; underline appears on hover.
.cta-textlink-purple
Purple — navigation-style links, footer links, in-page anchors, breadcrumbs. No underline at rest; underline appears on hover.

Font Awesome still works and there's no need to remove it from pages that already use it. But don't add it to new pages — use Hugeicons instead. Over time, migrate away from Font Awesome as you rebuild sections.

How to add in Webflow

Three methods — choose the one that fits your situation.

✓ Recommended · Method 1
Webflow App (no code)
Install the official Hugeicons app directly from the Webflow App Marketplace. Icons appear as native Webflow elements — resizable, styleable, and bindable to CMS.
1. Open Webflow Designer 2. Apps panel → search "Hugeicons" 3. Click Install → done
Method 2 · CDN font
Link tag in <head>
Add the CDN stylesheet to Site Settings → Custom Code → Head. Then use class names in any Code Embed block.
<!-- Site Settings → Head --> <link rel="stylesheet" href="https://cdn.hugeicons.com /font/hgi-stroke-rounded.css"> <!-- Anywhere on page --> <i class="hgi-stroke-rounded hgi-home-01"></i>

Inline SVG embed
Download the SVG from hugeicons.com and paste into a Code Embed block. Best when you need a single hero icon with specific colour or animation.
<!-- Webflow: Add element → Embed → paste SVG --> <svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor"> ...path data... </svg>
